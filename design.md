# PortfolioForge Design Direction

Use this guide for all user-facing UI and copy decisions.

## Brand Personality

- Calm confidence over hype.
- Practical and credible, not flashy.
- Friendly, human, and direct.
- Built for real freelancers and job-seekers.

## Voice Rules

- Write like a founder speaking to users, not a generic SaaS ad.
- Prefer concrete outcomes over abstract claims.
- Avoid stacked buzzwords and overpromising language.
- Keep sentences short and readable.
- Use plain English before marketing phrasing.

## Avoid These Patterns

- "AI-generated sounding" phrases like:
  - "unlock your potential"
  - "next-generation platform"
  - "revolutionary experience"
  - "without the studio bill"
  - "everything in one coherent flow"
- Overly symmetrical section structure with repetitive card copy.
- Too many gradients/shadows competing at once.

## Copy Style

- Lead with what users can do today.
- Mention constraints honestly (free vs paid) in straightforward terms.
- Use specific nouns: "portfolio link", "projects", "client inquiries", "applications".
- Prefer "Start free, upgrade when needed" tone.

## Layout Preferences

- One strong focal point per section.
- Keep visual rhythm varied (not every section same pattern).
- Use whitespace intentionally; don't fill every area.
- Let typography carry hierarchy before decorative effects.

## Components & Interaction

- Buttons: clear action labels (`Start free`, `View sample`, `Upgrade`).
- Cards: concise title + one practical sentence.
- Keep motion subtle and purposeful.
- Prioritize readability on mobile first.

## Pre-Ship Check

Before merging UI changes, verify:

1. Would this still read well with gradients removed?
2. Is every section saying something specific and useful?
3. Does it feel like a real product owner wrote it?
4. Would a skeptical client trust this page?
